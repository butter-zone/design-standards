# Design Tokens

Guidance for defining, naming, and using design tokens.

## Purpose

- Create a single source of truth for design decisions
- Enable consistency across platforms and surfaces
- Support automation and theming

## Token Principles

- Tokens represent intent, not values
- Names should describe usage, not appearance
- Tokens should be composable and scalable

## Example Categories

- Color
- Typography
- Spacing
- Motion
- Elevation
