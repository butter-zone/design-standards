# Accessibility Standards

Baseline accessibility principles for all design and implementation work.

## Goals

- Ensure inclusive, usable experiences by default
- Reduce retroactive accessibility fixes
- Provide clear, testable expectations

## Core Principles

- Accessibility is not optional
- Keyboard and screen-reader support is required
- Color contrast must meet WCAG guidelines
- Content must be perceivable, operable, and understandable

## Minimum Checklist (v1)

- Semantic HTML or equivalent structure
- Visible focus states
- Sufficient color contrast
- Meaningful labels and descriptions

## Future Additions

- WCAG mapping
- Component-level accessibility rules
- Testing guidance
